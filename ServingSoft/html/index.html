<strong><em>Описание проекта сервисного</em></strong>

CppCheck Misra Doxygen Stm32f407 CubeIde C11 FreeRTOS

Проект выполнен в среде разработки STM32CubeIde v. 1.8.0. Компилятор GCC C диалект gnu11. Целевой микроконтроллер stm32f407, плата stm32f4-Discovery. В проекте используются библиотеки CMSIS, HAL, LL и ОСРВ freeRTOS.

<em><strong>Выполняемые функции:</strong></em>

  • Управление парой бесколлекторных колёс (три обмотки и три датчика Холла)
  • Иформационное взаимодействие с Пультом через драйвер Bluetooth
  • Предоставление проекту регулятора API для управления
  • Распределение ресурсов МК, абстрагирование проекта регулятора от железа
  • Алгоритм управления уставкой для регулятора для:
    • поворота по/против часовой стрелки
    • смещения влево и вправо
    • смещение траектории катания вперёд и назад
    • увеличение/уменьшение амплитуды и периода

<em><strong>Алгоритм работы ПО</strong></em>

  1. Инициализация ядра МК
  2. Инициализация переферии МК
  3. Инициализация, настройка ОСРВ FreeRTOS
  4. Проверка причины перезапуска. Если был программный сброс из прерывания, то информирование этого события в Пульт ПК.
  5. Запуск планировщика ОСРВ

После этого логика проекта состоит из обработчиков прерывания и трёх потоков: Физический, Интерфейсный, Управленческий.

<em><strong>Обработчики прерывания</strong></em>

В преываниях аппаратных ошибок происходит программный перезапуск МК. Опрос датчиков Холла происходит в прерывании таймера 7 с частотой 100000 Гц. При изменении состояния датчиков Холла из прерывания происходит передача нового состояния в задачу Физического уровня. Обработчик прерывания таймера 2 сигнализирует потоку Управленческому, что надо вызывать функцию ТАУ проекта (регулятор). Прерывания используются для обмена информацией с Пультом. Преывания таймеров также используются для синусоидального режима управления колёсами и прочего.

<em><strong>Физический поток</strong></em>

  • Взаимодействие с прерыванием о событиях датчика Холла.
  • Расчёт угла поворота колёс.
  • Управление трёмя фазами для вращения колёс
  • Передача состояния колёс в потоки Интерфейса и Управленческий

<em><strong>Интерфейсный поток</strong></em>

  • Информационное взаимодействие с Пультом
  • Сбор информации о состоянии системы из потоков Физического и Управления.
  • Формирование пакета телеметрии
  • Выполнение команд с Пульта, передача соответствующих команд потокам Физическому и Управления

<em><strong>Поток управления</strong></em>

  • Вызов функции ТАУ проекта (регулятора)
  • Выполнение команд с Пульта
  • Передача информации о состоянии системы, связанной с управлением в потоки Физический и Интерфейсный
  • Предоставление API ТАУ проекту регулятора
  • Алгоритм управления уставкой для регулятора для:
    • поворота по/против часовой стрелки
    • смещения влево и вправо
    • смещение траектории катания вперёд и назад
    • увеличение/уменьшение амплитуды и периода

<em><strong>Стиль кодирования</strong></em>

  • Взаимодействие между потоками происходит через механизмы ОСРВ: очереди, мьютексы, семафоры, уведомления, буферы.
  • Взаимодействие через глобальные переменные сведено к минимуму.
  • Комментарии в стиле doxygen.
  • Обязательные требования misra 2012 учтены.
  • Статический анализ кода CppCheck.

<em><strong>Дополнительно</strong></em>

Регулятор вызывается из сервисного проекта, частота вызова настраивается через Пульт. По умолчанию частота 100 Гц. Траетория катания задаётся в сервисном проекте. На данный момент это синусоида. Возможно будет изменена траектория на 

const float k = DEFINE_COEF;
if (fmodf(t, period) < period/2.f)
{
  Q = k * fmodf(t, period)^2;
} else {
  Q = (k * (period/2.f)^2) - k * (fmodf(t-period/2.f, period)^2);
}
